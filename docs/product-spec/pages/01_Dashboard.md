# 01 Dashboard

## Page Purpose

Dashboard is the first page after opening the product.

It should help the user continue recent work quickly.

## User Goals

- Open a recent Project.
- Create a new Project.
- See device status summary.
- See recent scan activity.
- See product status.

## Main Content

Dashboard should include:

- Recent Projects
- Current Project summary
- Device status summary
- Recent Regions or Scans
- Quick actions

## Quick Actions

- New Project
- Open Project
- Open Last Project
- Check Devices

## What Should Not Appear

Dashboard should not show detailed scan parameters, serial commands, spectrum configuration, or raw logs.

## Layout Draft

```text
Header
Sidebar | Recent Projects | Current Status | Quick Actions
Status Bar
```

## Navigation

Dashboard can navigate to:

- Projects
- Devices
- Regions
- Scan
