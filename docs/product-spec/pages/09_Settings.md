# 09 Settings

## Page Purpose

Settings page manages system-level configuration.

## User Goals

- Manage device profiles.
- Manage default project path.
- Manage theme and language.
- Manage license information.
- Manage logs and diagnostics.

## Settings Sections

- General
- Device Profiles
- Data Paths
- UI Theme
- License
- Logs
- About

## Device Profiles

Device Profiles are reusable configurations.

A Project saves a Device Snapshot, but the source configuration is managed here.

## Rules

- Settings are system-level.
- Settings should not directly edit Project data unless explicitly imported into a Project.
- Advanced settings should be separated from normal workflow pages.
